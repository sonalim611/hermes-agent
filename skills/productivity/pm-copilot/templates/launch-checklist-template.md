## 1. Product readiness

- [ ] PRD / spec signed off
- [ ] Acceptance criteria verified in staging
- [ ] Edge cases documented
- [ ] Rollback criteria defined

## 2. Engineering readiness

- [ ] Code complete and merged
- [ ] Tests passing (unit, integration, smoke)
- [ ] Performance / load acceptable
- [ ] Feature flags configured
- [ ] Monitoring and alerts in place

## 3. Design and content

- [ ] UX copy reviewed
- [ ] Help docs or tooltips updated
- [ ] Release notes drafted

## 4. Go-to-market and comms

- [ ] Internal announcement scheduled
- [ ] Customer-facing comms drafted (if applicable)
- [ ] Support team briefed with FAQ

## 5. Legal, privacy, and compliance

- [ ] Privacy review (if PII touched)
- [ ] Terms / consent updates (if needed)
- [ ] Accessibility check (if UI changed)

## 6. Operations and support

- [ ] Runbook for on-call
- [ ] Support macros / ticket routing updated
- [ ] Success metrics dashboard live

## 7. Launch day

- [ ] Launch owner on point
- [ ] Rollout percentage / cohort confirmed
- [ ] War-room channel or standup scheduled
- [ ] Go / no-go decision recorded

## 8. Post-launch (first 48–72 hours)

- [ ] Metrics reviewed against success criteria
- [ ] Error rates and support volume monitored
- [ ] Retro scheduled
